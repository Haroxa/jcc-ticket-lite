# 10 系统信息与维护页

## 1. 页面目标

第一版不需要做复杂配置页。相比“改系统名称、改分页数量”这类低价值配置，更有用的是展示系统版本、部署状态和必要维护入口。

## 2. 布局

```text
┌──────────────────────────────────────────────┐
│ 系统信息                                      │
├──────────────────────┬───────────────────────┤
│ 版本介绍              │ 维护入口               │
│ 项目版本 v0.1.0       │ 查看操作日志           │
│ 目标平台 Cloudflare   │ 导出完整备份           │
│ 数据库 D1             │ 检查异常数据           │
│ 多端适配 已规划       │ 初始化管理员说明       │
└──────────────────────┴───────────────────────┘
```

## 3. 展示内容

版本介绍：

- 项目版本
- 当前环境：本地 / 预览 / 生产
- 目标平台：Cloudflare Workers + D1
- 数据策略：流水永久保存，作废不删除
- 多端支持：电脑、平板、手机

维护入口：

- 查看操作日志
- 导出完整备份
- 检查异常数据
- 初始化管理员说明
- 查看系统迁移状态

## 4. 暂不放入第一版的设置

- 自定义系统名称
- 自定义主题色
- 复杂角色权限矩阵
- 大量开关项

这些内容后续有明确需求再补。

# 11 移动端布局策略

## 1. 目标

手机上要能完成最常用操作：

- 查看余额
- 新增存入
- 新增取用
- 查个人历史
- 看最近记录
- 作废误录入记录
- 使用计算工具后回填票数

## 2. 导航

移动端使用底部导航：

```text
首页 | 录入 | 记录 | 存票人 | 更多
```

更多里放：

- 个人历史
- 导入导出
- 系统信息
- 退出登录

计算工具不放在更多里，直接位于快速录入页下方。

底部导航入口顺序保持稳定，其中 `录入` 放在第二位，方便高频操作。

## 3. 快速录入移动布局

单列布局：

```text
快速录入
日期
存票人搜索
当前余额
类型
票数
备注
保存按钮
最近记录
```

保存按钮固定在表单底部即可，不需要悬浮。

推荐尺寸：

```text
输入框高度 >= 44px
按钮高度 >= 44px
卡片间距 >= 10px
```

## 4. 表格移动适配

不要在手机上强行显示宽表格。

流水记录改成卡片：

```text
张三  存入 +100
2026-06-24  正常
备注：...
[作废]
```

卡片必须显示：

- 存票人
- 类型和票数
- 日期
- 状态
- 备注摘要
- 作废或恢复按钮

## 5. 汇总移动布局

统计块两列排列：

```text
当前总存票 | 存票人数
今日存入   | 今日取用
```

余额排行和最近记录上下排列。

## 6. 手机横屏

手机横屏仍按移动端处理，不切换为平板导航。

原因：

- 横屏高度不足，顶部横向导航会占用太多空间。
- 底部导航仍然更稳定。

## 7. 输入体验

- 数字字段唤起数字键盘。
- 搜索存票人时结果显示在输入框下方。
- 保存按钮宽度占满。
- 危险操作按钮不和主按钮贴太近。

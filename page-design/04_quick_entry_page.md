# 04 快速录入页

## 1. 页面目标

这是最高频页面。目标是让用户尽快完成存入或取用。

## 2. 桌面端布局

```text
┌────────────────────────────────────────────────────────┐
│ 快速录入                              当前日期 2026-... │
├──────────────────────────────┬─────────────────────────┤
│ 录入表单                      │ 当前存票人信息           │
│                              │                         │
│ 日期 [ 2026-06-24 ]           │ 张三                     │
│ 存票人 [ 搜索/选择 ]          │ 当前余额：1200            │
│ 类型 [ 存入 | 取用 ]          │ 最近记录：                │
│ 票数 [___________]            │ - 存入 100                │
│ 备注 [___________]            │ - 取用 50                 │
│                              │                         │
│ [保存记录] [清空] [跳到计算]  │                         │
└──────────────────────────────┴─────────────────────────┘
```

## 3. 字段

- 日期
- 存票人
- 类型
- 票数
- 备注

## 4. 交互规则

- 默认日期为当天。
- 存票人支持搜索。
- 类型使用分段按钮：`存入`、`取用`。
- 选择存票人后右侧展示当前余额。
- 选择取用时，票数超过余额要立即提示。
- 保存成功后：
  - 清空票数和备注。
  - 保留日期、存票人、类型。
  - 刷新右侧余额和最近记录。

## 5. 计算工具位置

计算工具放在快速录入页下方，而不是作为独立高频页面。

原因：

- 计算结果最终服务于录入票数。
- 用户不需要在两个页面之间反复切换。
- 手机端可以在录入表单下方继续向下滚动使用。

布局：

```text
快速录入表单
当前存票人信息

计算工具
常用价格 / 自定义价格 / 原始总数 / 抹零 / 取整 / 应用到票数
```

## 6. 错误提示

- 未选择存票人：`请选择存票人。`
- 票数为空：`请输入票数。`
- 票数不是正整数：`票数必须是大于 0 的整数。`
- 余额不足：`当前余额不足，不能取用。`

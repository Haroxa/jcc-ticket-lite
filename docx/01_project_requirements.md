# 01 项目需求与范围

## 1. 背景

当前 `存票管理系统_优化版_v9.xlsm` 已经具备较完整的存票管理能力，包括快速录入、存取记录、汇总看板、个人查询、数据维护和计算工具。

但 Excel 宏版不适合多人共享操作：

- 多人同时编辑容易冲突。
- 文件传来传去不利于统一数据。
- 权限控制弱。
- 操作日志和审计能力有限。
- 数据备份和长期维护依赖人工。

因此需要转换成一个轻量 Web 项目，部署到 Cloudflare，并使用 Cloudflare D1 持久化保存数据。

## 2. 项目定位

本项目是 `JCC Table` 的轻量 Web 转换版，不与 `JCC-Web-MVP` 混合。

它只聚焦存票管理本身，尽量快速实现、快速上线、快速使用。

## 3. 核心目标

- 支持网页登录使用。
- 支持新增存入和取用记录。
- 支持按存票人汇总当前余额。
- 支持查看单个存票人的历史记录。
- 支持作废记录，不直接删除正式记录。
- 支持基础权限：管理员、录入员、查看者。
- 支持操作日志。
- 支持从 Excel v9 导入历史数据。
- 支持部署到 Cloudflare。
- 支持数据永久保存到 Cloudflare D1。

## 4. 第一版 MVP 范围

第一版必须包含：

- 登录页
- 首页汇总
- 快速录入
- 存票人管理
- 存取记录列表
- 个人历史查询
- 作废记录
- CSV 导入导出
- 基础设置

第一版暂不包含：

- 复杂主播/场次/比赛管理
- 截图上传
- 排名池
- 大型多模块后台
- 多租户商业化能力
- 复杂审批流

## 5. 角色设计

### 管理员

- 管理账号。
- 管理存票人。
- 新增存入和取用记录。
- 作废和恢复记录。
- 导入导出数据。
- 查看操作日志。
- 修改系统设置。

### 录入员

- 新增存入和取用记录。
- 查看存取记录和汇总。
- 不能删除数据。
- 可申请或执行作废，具体权限可后续配置。

### 查看者

- 只读查看汇总和历史。
- 不能新增、修改、作废、导入。

## 6. 关键业务规则

- 正式记录不物理删除，只允许作废。
- 作废记录不参与余额汇总。
- 取用票数不能超过当前余额。
- 票数必须为正整数。
- 存票人名称不建议重复。
- 每次新增、作废、恢复、导入都写入操作日志。
- 汇总余额以流水记录为准，可额外缓存余额提升查询速度。

## 7. 成功标准

- 用户可以在浏览器完成 v9 的主要存票流程。
- 多人访问时数据统一保存在云端。
- 刷新页面、换设备、重新登录后数据仍在。
- 数据可导出备份。
- 历史 Excel 数据可以迁移进去。

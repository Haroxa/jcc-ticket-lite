## 11. 风险与优先级

高优先级风险：

- 余额计算错误。
- 公开接口泄露内部字段。
- 停用或拉黑人仍可被录入。
- 远程 D1 未迁移导致线上报错。
- Session Secret 未配置导致登录异常。

中优先级风险：

- 移动端表格展示拥挤。
- 操作日志筛选不够清楚。
- 导入数据格式不统一。

低优先级风险：

- 系统信息页面内容偏少。
- 导出格式需要后续微调。

## 12. 建议实施节奏

第一轮先完成：

```text
项目骨架 -> D1 建表 -> 登录 -> 存票人 -> 快速录入 -> 记录查询 -> 公开榜
```

第二轮再完成：

```text
个人历史 -> 操作日志 -> 导入导出 -> 设置页完善 -> 多端体验打磨
```

这样可以最快得到一个可部署、可真实录入、可永久保存的版本。

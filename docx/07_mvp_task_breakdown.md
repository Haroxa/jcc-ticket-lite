# 07 MVP 实施拆解

## 阶段 1：项目初始化

- 创建 Vite React TypeScript 项目。
- 创建 Worker API 目录。
- 配置 Wrangler。
- 创建 D1 数据库。
- 创建第一版迁移 SQL。

验收：

- 本地页面能打开。
- `/api/health` 返回正常。
- 本地 D1 可执行迁移。

## 阶段 2：账号与登录

- 建立账号表。
- 实现管理员初始化。
- 实现登录。
- 实现退出。
- 实现当前用户接口。
- 前端增加登录页。

验收：

- 未登录不能进入业务页。
- 管理员可以登录。
- 退出后需要重新登录。

## 阶段 3：存票人管理

- 建立存票人表。
- 实现新增、编辑、停用。
- 实现搜索。
- 显示当前余额。

验收：

- 可以新增存票人。
- 可以搜索存票人。
- 停用存票人不会出现在快速录入默认列表。

## 阶段 4：存取记录

- 建立流水表。
- 实现新增存入。
- 实现新增取用。
- 实现余额不足校验。
- 实现作废和恢复。
- 刷新缓存余额。

验收：

- 存入后余额增加。
- 取用后余额减少。
- 余额不足不能取用。
- 作废后余额恢复。
- 恢复后余额重新变化。

## 阶段 5：首页与个人历史

- 首页汇总接口。
- 余额排行。
- 最近记录。
- 个人历史接口。
- 前端页面展示。

验收：

- 首页数字正确。
- 点击存票人可看到历史。
- 筛选日期后结果正确。

## 阶段 6：计算工具

- 实现常用价格配置。
- 实现加减价格。
- 实现自定义价格。
- 实现向下抹零和向上取整。
- 实现应用到快速录入。

验收：

- 与 v9 计算工具行为一致。

## 阶段 7：导入导出

- CSV 导出全部记录。
- CSV 导出余额表。
- CSV 导入预览。
- CSV 导入提交。
- 导入日志。

验收：

- 能从 v9 导出的 CSV 导入。
- 导入后余额能对上。
- 错误行不会进入正式数据。

## 阶段 8：部署

- 配置远程 D1。
- 运行远程迁移。
- 部署到 Cloudflare。
- 设置初始化管理员。
- 完成线上验收。

验收：

- 线上地址可访问。
- 数据刷新后仍在。
- 不同设备登录后看到同一套数据。

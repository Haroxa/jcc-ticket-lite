# 06 从 Excel v9 迁移计划

## 1. 迁移来源

来源文件：

```text
JCC Table/存票管理系统_优化版_v9.xlsm
```

重点工作表：

- 存取记录
- 设置
- 汇总看板

## 2. 迁移目标

把 Excel v9 中的存票人、存取流水和常用价格迁移到 Cloudflare D1。

## 3. 建议迁移字段

从 `存取记录` 迁移：

| Excel 字段 | 新系统字段 |
|---|---|
| 记录ID | ticket_records.id 或 legacy_id |
| 日期 | business_date |
| 存票人 | ticket_people.name |
| 类型 | type |
| 票数 | amount |
| 库存变化 | balance_delta |
| 状态 | status |
| 备注 | note |
| 录入时间 | created_at |

如果 Excel 中没有稳定记录 ID，则导入时生成新 ID，并保留一列 `legacy_row_no`。

## 4. 迁移流程

1. 在 v9 中执行数据检查。
2. 从 v9 导出存取记录为 CSV。
3. 在新系统中上传 CSV。
4. 系统进入导入预览。
5. 自动识别存票人。
6. 自动校验字段：
   - 日期是否有效
   - 类型是否为存入/取用
   - 票数是否为正整数
   - 状态是否有效
   - 库存变化是否匹配类型
7. 显示错误行和警告行。
8. 管理员确认导入。
9. 写入存票人和存取记录。
10. 重新计算所有存票人余额。
11. 生成导入日志。

## 5. 导入校验规则

错误：

- 存票人为空
- 类型为空或不识别
- 票数不是正整数
- 日期无法识别

警告：

- 重复记录 ID
- 同一存票人同一天同类型同票数疑似重复
- 状态为空，默认按正常处理
- 备注过长

## 6. 迁移后的验收

需要核对：

- 总存票是否与 v9 汇总一致。
- 每个存票人的余额是否一致。
- 存入合计是否一致。
- 取用合计是否一致。
- 作废记录是否不参与汇总。
- 随机抽查 5 到 10 个存票人的历史记录。

## 7. 回滚策略

第一版导入前建议：

- 先在本地 D1 测试导入。
- 确认无误后再导入远程 D1。
- 远程导入前导出现有数据备份。
